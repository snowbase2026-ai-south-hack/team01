## Architecture

Monolith FastAPI app (`main.py`) that serves as an LLM-powered strategic decision assistant. All logic in one file.

**Request flow:**
1. HTTP POST → one of 5 chat endpoints → `process_chat()`
2. Input validation + sanitization + prompt injection detection
3. Conversation history retrieved from in-memory `ConversationStore`
4. LLM call via OpenRouter API (OpenAI-compatible client) with system prompt + history
5. Response stored in session, returned as JSON (all field aliases: response/answer/message/content/text)

**Key components in `main.py`:**
- `SYSTEM_PROMPT` (~276 lines) — full case data, role behavior rules, scenario analysis, stakeholder map
- `ConversationStore` — in-memory session dict, 40 messages max per session
- `sanitize_input()` / `detect_injection()` — input safety layer
- `extract_message()` — handles message/query/messages/text/content/input/prompt fields
- `process_chat()` — core logic, supports streaming (SSE) and non-streaming modes
- 5 POST endpoints: `/api/chat`, `/api/v1/chat`, `/chat`, `/api/message`, `/api/query`
- Global exception handler catches all errors → returns 400 (never 500)

**LLM backend:** OpenRouter API (`openai` SDK with custom base_url). Model configurable via `OPENROUTER_MODEL` env var. Default: `anthropic/claude-3.5-haiku`.

## API Contract (hackathon requirements)

- **Primary endpoint:** `POST /api/chat` with `{"message": "..."}` → `{"response": "..."}`
- Also accepts: `query`, `messages` (OpenAI-style), `text`, `content` fields
- All 5 endpoint paths return identical responses
- `GET /health` → `{"status": "ok"}`
- `GET /docs` → Swagger UI (auto-generated by FastAPI)
- CORS enabled for all origins
- Streaming via `{"stream": true}` → SSE
- Session management via `session_id` field, reset via `POST /api/reset`

**Error handling rules (critical for scoring):**
- Empty body → 400
- Invalid JSON → 400
- Missing message field → 400
- Unknown path → 404
- **Never return 500** — global exception handler catches everything

## Scoring Criteria

**AutoScore (70%):** management logic (50pts), functionality & data (20pts), security (10pts), stability & UX (15pts), cost (5pts).

**Pitch (30%):** product usefulness, system quality, architecture & assumptions, pitch quality. Scale 0/1/2.

**Stress test:** 5 waves — (1) CEO+CDTO emotional pressure, (2) CFO CAPEX -30%, (3) COO SLA drop, (4) ML team degradation data, (5) CDTO leaves company.
